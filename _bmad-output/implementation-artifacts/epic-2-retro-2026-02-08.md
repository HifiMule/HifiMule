# Retrospective: Epic 2 - Connection & Verification (The Handshake)

**Date:** 2026-02-08
**Project:** HifiMule
**Epic:** 2 - Connection & Verification (The Handshake)

## Summary
Epic 2 focused on the secure handshake between the user, the device, and the Jellyfin server. This included OS-native credential storage, automated USB device detection, unique device identification, and a coordinated startup splash screen.

## Delivery Metrics
- **Completed:** 4/4 Stories (100% velocity)
- **Primary Deliverables:** Secure Keyring storage, `MountObserver` for USB events, SQLite-backed `DeviceManager`, and a Glassmorphism Splash Screen.
- **Memory Footprint:** Maintained under the 10MB goal (currently ~3.15MB private bytes on Windows).

## Successes (What Went Well)
- **Robust Multi-Platform detection:** The `MountObserver` abstraction works seamlessly across Windows bitmasks and Unix volume scans.
- **Zero-Config UX:** Plugging in a known device now automatically identifies the Jellyfin profile via SQLite, fulfilling the "Ritualist" user persona's core needs.
- **Splash Screen Polish:** The Tauri-native splash screen significantly improves perceived performance and app stability during cold starts.
- **Security Burn-in:** Successfully transitioned from plain JSON credentials to OS-native `keyring` storage.

## Challenges (What Was Difficult)
- **Windows Keyring Persistence:** Spent significant effort debugging `keyring-rs` behavior on Windows where credentials would seemingly disappear due to Entry instance scope issues.
- **CORS / RPC Security:** Balancing the need for local IPC with browser security constraints in Tauri (fixed via strict origin filtering).
- **Time-Based State Races:** Early implementations of the event loop had fragile time-resets; refactored to a more robust state machine in `main.rs`.

## Lessons Learned
- **Daemon-as-Abstraction:** Building media-server-specific logic into the daemon (proxying) is superior to direct UI-to-Server communication for future scalability (Plex/Emby support).
- **Windows Native APIs are "Chatty":** Polling logical drives requires careful debouncing to avoid CPU spikes.
- **Shared Path Utilities:** Moving AppData logic to a dedicated `paths` module saved significant refactoring time.

## Action Items
- [ ] **Implement RPC Proxying:** Extend the daemon's RPC layer to proxy Jellyfin metadata, removing server-specific logic from the UI.
